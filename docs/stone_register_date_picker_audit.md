# Stone Register Date Picker Migration Matrix

The HTML mockup is a skin and interaction reference only. It does not define
canonical month data, conversion rules, persistence formats, validation, route
behavior, or date-only semantics.

The shared picker was introduced first, followed by low-risk reusable wrappers
and bounded wrapper consumers. Complex calendar, event, recurrence, Day View,
Ma'at flow, and journal navigation surfaces remain deferred until their
preservation contracts are written and tested.

### Flow Studio Start/End Preservation Contract

- Entry points: `_pickGregorianDate`, `_pickKemeticDate`, `_pickRangeStart`, `_pickRangeEnd`, `_pickComposeRangeStart`, and `_pickComposeRangeEnd` in `lib/features/calendar/calendar_flow_studio_page.dart`.
- Current public/local API: private picker helpers return `Future<DateTime?>`; build and compose callers own all state mutation after a non-null result.
- Date model and defaults: all returned values are Gregorian date-only `DateTime`; picker seeds from the current selected value or `DateTime.now()`; build end defaults to `_endDate ?? _startDate`; compose end defaults to `_composeEndDate ?? _composeStartDate`.
- Display formatting: build and compose labels continue through `_fmtGregorian` and `_fmtKemetic`; Kemetic month names continue through `getMonthById`.
- Validation: Gregorian month/leap-day clamping, Kemetic epagomenal 5/6 clamping, and existing `_rebuildSpans` start/end order normalization must remain unchanged.
- Persistence path: date picking only mutates local Flow Studio state; draft persistence, `_FlowStudioResult`, AI import initialization, planned-note payloads, and calendar scheduling remain caller-owned.
- Cancel/Done/reopen: Cancel returns null and does not mutate build or compose ranges; Done saves exactly the visible selected date through existing caller normalization; reopening shows the saved selected value.
- Dirty state and side effects: build date changes keep `_dateRangeEditedInCurrentEditor` and `_applySelectionToDrafts`; compose date changes keep `_composeManualDateRangeEdited`, duration fallback, and `dateOnlyForAiFlow`; AI generation still uses `resolveAiFlowDateRange`.
- Existing tests: `test/features/calendar/flow_studio_state_sync_regression_test.dart` already covers draft state, range labels, system toggles, generated save payloads, import initialization, route results, generated schedule spans, and mobile toolbar layout. This migration must add focused picker tests for open, Cancel, Done, reopen, range validation preservation, Compose behavior, no off-by-one dates, and small-phone overflow.

## Verification Log

- Shared picker tests cover open/cancel/done, mode switching, Gregorian leap/month behavior, migrated wrapper done behavior, and Kemetic `KemeticMath`/`getMonthById` mapping.
- AI modal wrapper-consumer tests cover Gregorian wrapper open/cancel/done through the real start-date button, Kemetic wrapper open/done through the real start-date button, small-phone layout, and duration preservation.
- Birthday editor tests cover required-field validation, small-phone picker cancel without mutation, picker done/save/reopen date-only behavior through `createBirthday`, and Feb. 29 support in the bounded Gregorian adapter.
- Flow start picker tests cover existing-value open, small-phone cancel without mutation, done/reopen date-only behavior, default-tomorrow seed, 40-year Gregorian window, and Kemetic selection round-trip.
- Node user insight consumer tests cover small-phone picker open, Kemetic month display, cancel without date/body mutation, done/reopen behavior, and editor body preservation without touching insight persistence.
- Shared flow import consumer tests cover suggested-date fallback label, small-phone picker open, cancel without mutating the footer, done/reopen behavior, and import CTA preservation without invoking import or scheduling side effects.
- Saved flow start consumer tests cover past-start normalization to today, small-phone picker open, cancel without mutating the footer, done/reopen behavior, and Import Flow CTA preservation without invoking import or scheduling side effects.
- Flow Studio start/end tests cover Gregorian build picker open/cancel/done/reopen on small phone, Kemetic conversion display through app calendar utilities, compose picker cancel/done/reopen with prompt preservation, wrapper routing, range normalization guard, generated schedule/import regression coverage, and AI generation date wrapper smoke coverage.
- Remaining complex surfaces were intentionally not migrated in this pass.
